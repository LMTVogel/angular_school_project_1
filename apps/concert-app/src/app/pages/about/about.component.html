Over dit project

Casusbeschrijving en functionaliteiten

Uiteindelijk moet dit een website worden waar je als gebruiker concerten kan zien en dan kun je hier een kaartje voor kopen. Je kan dan zien waar het zich afspeelt en wie er komen optreden. Functionaliteiten: User mag alleen zijn eigen gekochte kaartjes inzien. User mag wel alle concerten zien en daar een kaartje voor aanschaffen. User moet details van een concert kunnen zien. User moet zijn eigen aangeschafte kaartjes in kunnen zien. User moet kunnen inloggen en user moet kunnen registreren uiteraard.

Admin moet CRUD functionaliteiten hebben op alle concerten. Admin moet ook alle users op kunnen vragen.

Als user1 een kaartje heeft voor concert1 en user2 heeft ook een kaartje voor concert1 heeft, dan hebben ze waarschijnlijk dezelfde interesse. User2 heeft ook een kaartje voor een ander concert en waarschijnlijk vindt user1 dit ook interessant. Daarom gaan we dit aan user1 aanraden omdat ze misschien overeenkomende interesse hebben.

Entiteitsrelaties

Een user mag meerdere tickets aanschaffen. Een ticket kan maar bij 1 user horen. Concert heeft meerdere tickets en vindt plaats op 1 locatie. Op dit concert zijn 1 of meerdere artiesten die optreden. Een artiest kan bij nul of 1 band zitten en een band heeft meerdere artiesten.
